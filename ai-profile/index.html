## 3. 回覆與合作偏好

其他 AI 與盧老師互動時，建議遵守以下偏好：

1. **使用繁體中文。**
2. **直接、低摩擦、可執行。**  
   不要只給抽象建議；能做就直接做，能產出就產出。
3. **不需要過度解釋基本物理。**  
   盧老師可理解大學程度物理；若是為學生製作教材，再轉成教學語言。
4. **偏好實作導向。**  
   例如：腳本、Markdown、簡報、網站、影片、程式碼、流程圖、檢核表、可複製 prompt。
5. **偏好穩定成果。**  
   正式交付應使用 GitHub / GitHub Pages 穩定連結、Obsidian 檔案或 Telegram 附件；避免臨時網址。
6. **語氣風格：溫和、務實、謙遜。**  
   避免過度宣傳語、過度煽情標題、浮誇權威感。
7. **如果是多步驟任務，請主動拆解並執行。**
8. **若使用外部資料，請盡量附可查證來源。**
9. **若涉及學生資料、課堂紀錄或推薦信，需注意個資與尊嚴。**

## 4. 推薦信與正式文字風格

盧老師常需要撰寫：

- 語氣可以正式，但不要官腔。
- 盧老師通常以「物理老師」、「TYPT 培訓老師」、「指導老師」等身份書寫。

## 5. 影片與 YouTube Shorts 工作流

### 5.1 主要方向

盧老師正在發展教育型 YouTube / Shorts 內容，方向包括：

- AI 教育。
- AI 進教室前教師應知道的小事。
- AI 輔助課堂摘要、批改、學生回饋。
- 學生資料隱私與倫理。
- 物理分科測驗詳解。
- 每日一題物理解析。
- 探究與實作、物理教學、AI 工具使用。

### 5.2 推薦影片定位

影片不應只是「工具展示」，而應兼具：

1. 教學現場感。
2. 可複製工作流。
3. 風險提醒。
4. 盧老師的溫和教育視角。

近期建議主軸：

> AI 進教室，不是取代老師，而是讓老師把時間拿回來；但每一步都要守住學生尊嚴。

### 5.3 Shorts 常用結構

建議 30–60 秒 Shorts 可使用：

1. **Hook：** 一句讓老師停下來看的問題。
2. **錯誤示範：** 一個真實課堂語境中的錯誤句子或錯誤操作。
3. **風險標示：** 個資、標籤化、主觀推測、AI 幻覺等。
4. **改寫示範：** 改成安全、可行動、支持學習的版本。
5. **收束金句：** 溫和但有記憶點。

範例金句：

> 記錄學習困難，不標籤學生。  
> 摘要服務學習，不是標記學生。  
> AI 可以幫老師省時間，但不能替學生貼標籤。

### 5.4 物理教學影片風格

針對物理題目詳解，偏好：

- 先確認物理正確性。
- 使用「直覺層 → 推導層 → 考場層」。
- 視覺應服務理解，不只是裝飾。
- 公式逐步出現，不要一次塞滿。
- 強調考場判斷、常見陷阱與模型選擇。
- 音訊與視覺提示需同步。

### 5.5 聲音與旁白

- 所有 narration / voice / audio / 旁白，**預設使用盧老師 voice clone**。
- 使用者曾比較過 F5-TTS 與 CosyVoice2，偏好 **F5-TTS 版本**。
- 若無法使用盧老師 voice clone，應先回報，不要默默改用其他聲音。

## 14. 學生與教育倫理注意事項

盧老師重視學生尊嚴與學習支持。處理學生相關文字時，AI 應注意：

- 不暴露姓名、座號、可識別資訊，除非盧老師明確提供且用途合理。
- 不使用羞辱性、貶抑性、人格化標籤。
- 不把「一次表現」寫成「人格特質」。
- 不推測學生動機，例如「懶惰」「不想學」「沒在聽」，除非有明確證據且以教育支持語言重寫。
- 課堂摘要應聚焦：
  - 學習困難。
  - 可觀察證據。
  - 下一步支持。
  - 教學調整。

可用原則：

> 寫學習證據，不寫人格判斷。  
> 記錄困難，不標籤學生。  
> 摘要服務學習，不是標記學生。

## 15. 對其他 AI 的快速操作建議

如果你是另一個 AI，請依任務類型這樣做：

### 15.1 寫正式文字 / 推薦信

1. 保持真誠、具體、教師觀察感。
2. 用繁體中文。
3. 避免浮誇。
4. 盡量保留學生具體事例與成果。
5. 若缺少重要資訊，可先給一版可用草稿，再列出可補充欄位。

### 15.2 做影片 / Shorts

1. 先寫 30–60 秒腳本。
2. 再做 storyboard。
3. 視覺要能教學，不只是好看。
4. 預設使用盧老師 voice clone。
5. 交付穩定 MP4、GitHub Pages 連結或 Telegram 附件。

### 15.3 做網站或公開成果

1. 使用穩定 GitHub Pages。
2. 不要用臨時網址作正式交付。
3. 修改後提供公開連結與 before / after 效果。
4. 若在 repo 中作業，請檢查 git status、commit、push。

### 15.4 做 Obsidian / 筆記整理

1. 先讀 `AI_CONTEXT.md`、`HANDOFF.md`。
2. 新內容優先 Markdown。
3. 不確定分類就放 `Inbox/Agent Inbox/`。
4. 重要成果更新 Daily 或相關總覽。

### 15.5 做物理教學內容

1. 先驗證物理正確性。
2. 再轉成教學腳本。
3. 強調模型選擇、直覺、推導與考場辨識。
4. 不要只列公式。
