Overview

不是聊天框，而是可上課的 AI 教室包

OpenMAIC 的精神是讓 LLM agents 組成像真實教室一樣的社會學習環境。這個繁中物理版把它縮成老師可直接展示、可編輯、可匯出的單頁工作台。

從主題描述到完整課堂

輸入年級、時間、主題、學生樣貌與迷思後，系統分兩階段工作：先產生課程大綱，再把每個大綱項目轉成課堂場景：白板、活動、評量、模擬與學生講義。

本頁預設使用安全的前端模板生成；若啟用真 AI，前端只呼叫本機/伺服器代理，代理端再使用 Hermes OpenAI Codex OAuth，避免把 OAuth token 或 API key 放在 GitHub Pages。

Architecture

需要 API 時：改走盧老師的 Codex OAuth

真正的 Codex OAuth token 留在 WSL 的 Hermes，不放到瀏覽器。公開頁面只送課程需求到代理端，代理端用 hermes chat --provider openai-codex -m gpt-5.5 生成 JSON，再回傳給頁面。

安全資料流

FRONTENDGitHub Pages 靜態頁只保存課程欄位與 UI 狀態，不保存任何 key/token。
LOCAL PROXYHermes Codex OAuth Adapter本機或受控伺服器接收 prompt，呼叫 Hermes CLI。
LLM PROVIDERopenai-codex / gpt-5.5使用盧老師已登入的 OAuth，不需要公開 OpenAI API key。
FALLBACKClient-side deterministic template代理不可用時，仍能在手機上展示完整互動原型。
# proxy 端核心概念：OAuth 留在 Hermes
hermes chat \
  --provider openai-codex \
  -m gpt-5.5 \
  -q "依照輸入 JSON 產生 OpenMAIC-style 高中物理 course package，輸出 JSON" \
  -Q

# 前端只呼叫代理，不帶密鑰

FAQ

常見問題

這一版真的有接 Codex OAuth 嗎？

公開 GitHub Pages 不能直接存 OAuth，這是不安全也不可行的。本頁已把 API 設計改成「前端 → Hermes Codex OAuth 代理 → Codex」的架構；若代理沒開，會自動回到前端模板。這樣展示頁可公開，真 AI 可在盧老師自己的 Hermes 環境執行。

為什麼不像直接放 OpenAI API key？

因為瀏覽器中的 key 會被任何人看見。Codex OAuth 也更不應暴露。正確做法是後端代理或 Hermes gateway 代呼叫。

這適合真實課堂嗎？

適合當備課與共備原型：老師先生成課程包，再根據班級狀態修訂。物理正確性仍應由老師最後把關。
