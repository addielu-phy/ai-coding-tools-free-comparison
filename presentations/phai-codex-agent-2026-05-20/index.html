開場語氣：不要把自己放成「大師」，而是「我這幾週把 AI 當成一群助教來磨，今天分享哪些真的省下時間、哪些還需要老師把關」。
今晚 60 分鐘不是工具展示，是工作流展示

我想回答三個問題

1

Agent 到底幫老師做什麼？

不是聊天，而是把任務拆成可交付的工作包。

2

物理教學如何落地？

從題目來源、建模、腳本、圖卡、旁白、字幕到 QA。

3

老師怎麼守住品質？

物理正確性、學生尊嚴、引用授權、可追溯紀錄。

提醒大家：今晚不是要大家馬上用同一套工具，而是看「把任務交給 AI 團隊」的設計方法。
核心比喻從單一 Chatbot 到助教團隊

以前我問 AI：「幫我做教材」
現在我派工：「誰負責哪一段、如何驗收」

Chatbot

一次問答，靠老師記得所有脈絡。

→

Agent

有工具、有檔案、有驗證，能完成一個小任務。

→

Agent Team

多個角色協作，產出可追蹤、可重做、可交接的成果。

這張用生活化方式講：不是 AI 比較聰明，而是「任務管理方式」變了。老師從逐字指揮改成定義目標、限制、驗收標準。
我的 AI 教學助理團隊角色不是工具名，而是責任

一個教學任務，可以拆成六個助教角色

盧老師

決定教學意圖
把關物理與倫理

資料偵察員

找來源、整理題目、標注引用風險

物理審稿員

驗算答案、找易錯點、確認模型

教材編劇

hook、分層說明、講稿與 prompt

視覺導演

storyboard、動畫 cue、圖卡排版

製作助理

旁白、字幕、封面、upload kit

QA／知識庫管家

尺寸、可讀性、檔案清單、Obsidian 交接

可以說：我不會把「判斷學生」交給 AI；但我會把「整理材料、初稿、檢查清單」交給 AI。
工作流地圖老師的時間花在最值得花的地方

每個成果都走同一條「可驗收」管線

1. 捕捉

Telegram／Obsidian 收想法、材料、題目。

→

2. 拆工

把任務拆成 brief、storyboard、QA、交付包。

→

3. 產出

HTML、Markdown、MP4、字幕、封面、程式。

→

4. 驗證

物理、版權、視覺、檔案、GitHub／附件。

重點不是「AI 幫我想」，而是「AI 幫我把可重複的教學工序做成管線」。

給老師的 Prompt 模板可直接複製

把「幫我做教材」改成「請產出可驗收交付包」

你是我的物理教學助理。請不要只回答概念，請做成可交付的教學工作包。

任務：把【主題／題目】做成一節課可用的素材。
對象：【高一／高二／分科考生／師培生】
請輸出：
1. 教學目標與學生可能卡點
2. 核心模型或公式，但先用直覺語言說明
3. 5 分鐘課堂引導流程
4. 1 張板書／投影片草圖
5. 2 題形成性評量
6. 老師檢查清單：物理正確性、學生迷思、資料來源、倫理風險

限制：不要捏造來源；不確定請標「需老師確認」。
常見踩雷我的修正方式

AI 助教團隊也需要班級經營

容易踩雷

  • AI 產出看起來完整但未驗算
  • 漂亮圖卡塞太多字
  • 用臨時網址當正式交付
  • 忘記來源與題文引用風險
  • 讓 AI 對學生做標籤式判斷

修正方法

  • 每題保留 build script / manifest / QA
  • 一張圖只講一個判斷
  • 正式成果用 GitHub Pages 或附件
  • 公開前設人工 gate
  • AI 只整理行為證據，不做人格判定
收束把時間還給學生

AI 不取代老師，
它替老師搬走可規格化的重工。

老師保留

教育判斷、關係、價值選擇、最後把關。

Agent 承擔

整理、轉寫、格式化、檢查、交付、記錄。

學生得到

更快回饋、更清楚材料、更有品質的學習時間。

最後回到活動文案：把繁瑣備課交給 AI 團隊，把時間還給學生。語氣溫和，不要像賣工具。
